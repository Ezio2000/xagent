# Security Policy

Use GitHub private vulnerability reporting for `Ezio2000/jharness` when a schema,
portable behavior, or specification-level ambiguity could cause unsafe execution or
data exposure. Do not publish credentials, user data, or exploit details in an issue.

Implementation-specific vulnerabilities belong in the affected implementation
repository:

- Python: `Ezio2000/jharness-python`;
- Go: `Ezio2000/jharness-go`.

Released specification tags are immutable. A security correction is published as a
new specification version and adopted explicitly by each affected implementation.
